# Stitch AI Revision Prompt - Make It Desktop Web

Use this as a follow-up prompt in Stitch AI if it starts generating mobile-app-like screens.

---

Please revise the generated UI direction. The current output is too mobile-app oriented. I need a desktop-first web SaaS interface, not a mobile app.

Important correction:

- Do not use phone-sized screens as the primary output.
- Do not use iOS/Android style bottom navigation.
- Do not use mobile app frames.
- Do not create a native mobile LMS app.
- This product is a web-based platform for browser use.

Please redesign the screens as wide desktop browser layouts:

- Public landing page as a full-width website.
- Entry point as a desktop web page with choice cards.
- Owner dashboard as a SaaS admin dashboard with sidebar navigation, topbar, metric cards, tables, and activity panels.
- Course detail as a desktop page with tabs: Overview, Lessons, Classes, Settings.
- Class/group detail as a desktop page with tabs: Overview, Students, Homework, Access.
- Lesson editor as a desktop form/editor with sections: Basic Info, Video, Materials, Homework, Quiz/Test, Publish Rules.
- Homework review as a mentor inbox with a left filtered submissions list and a right detail/review panel.
- Student dashboard as a web portal page, not a phone app. It can be simpler than the admin dashboard, but still a browser page.
- Public school/course pages as website pages with lead forms.
- Superadmin organization creation as an admin web form with a preview panel.

Use desktop web components:

- Top navigation
- Sidebar navigation
- Dashboard metric cards
- Data tables
- Filter bars
- Tabs
- Split panes
- Form sections
- Status badges
- Empty/loading/error states

Keep the visual direction:

- Trustworthy
- Organized
- Modern
- Calm
- Education SaaS
- Operational dashboard

Keep all product decisions:

- Multi-tenant schools/organizations
- Superadmin-controlled school creation
- Public school request form
- Demo school mode
- Course = reusable content package
- Class/group = live cohort operation
- Class-level access control
- Telegram stream video references
- Phone + password login, no SMS OTP, no Telegram login

Please regenerate the design as a desktop-first web app UI system.
